# 2026-04-25 工作记录

## 微信开发者工具加载适配

修复了阶段A代码在微信开发者工具中无法加载的问题：

1. **核心修复：shared 目录复制到 miniprogram 下**
   - 原先 `api.ts` 引用 `../../cloudfunctions/shared/`，开发者工具编译 miniprogram 时不会包含 cloudfunctions 目录
   - 将 `cloudfunctions/shared/` 复制到 `miniprogram/shared/`，前端引用本地副本
   - `api.ts` 的 import 路径从 `../../cloudfunctions/shared/` 改为 `../shared/`

2. **project.config.json 配置补全**
   - 添加 appid: `wxd45d5c3e7b56e89a`（测试号）
   - 添加 packOptions.ignore: 忽略 cloudfunctions、scripts、tsconfig.json
   - 创建 project.private.config.json（避免开发者工具初始化提示）

3. **game.ts 事件接收 Bug 修复**
   - action-sheet 组件通过 `triggerEvent('select', { action })` 触发事件
   - game.ts 的 onAction 应读 `e.detail.action`，原来错误读取 `e.currentTarget.dataset.action`
   - 修复为 `e.detail?.action || e.currentTarget.dataset.action`（兼容两种方式）

4. **miniprogram/package.json 创建**
   - 微信开发者工具"构建 npm"功能需要此文件

5. **编译验证**
   - TypeScript 零逻辑错误通过

6. **运行时超时修复**
   - 错误：`Error: timeout at WAServiceMainContext.js`
   - 原因1：`useCompilerPlugins` 为 `false`，开发者工具未启用 TS 编译插件，生成了默认模板 JS 覆盖了我们的代码
   - 原因2：`cloudfunctionRoot` 配置存在但云环境未开通，开发者工具尝试初始化云服务超时
   - 修复：`useCompilerPlugins` 改为 `["typescript"]`，删除所有自动生成的默认模板 `.js` 文件，移除 `cloudfunctionRoot`
   - 注意：微信 TS 编译使用 `@babel/plugin-transform-typescript`，只做类型擦除不做模块解析，但会递归编译被引用的 TS 文件

7. **真机调试 WXSS 中文类名报错修复**
   - 错误：`wxss 编译错误: unexpected char at action-sheet.wxss(1:561)`
   - 原因：WXSS 中 `.action-硬扛推进` 等中文类名在真机编译器中不兼容
   - 修复：所有中文分类 class 改为英文映射（硬扛推进→force, 证据管理→evidence 等9个）
   - 组件侧新增 `CATEGORY_CLASS_MAP` 映射表 + `observers` 将 `options.category` 转为 `cssClass`
   - WXML 遍历改为 `mappedOptions`，class 拼接改为 `action-{{item.cssClass}}`
