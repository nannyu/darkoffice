# 核心循环

## 1. 文档目的

本文档定义《暗黑职场》的基础玩法循环，回答玩家每回合在经历什么、系统如何施压、玩家如何做取舍，以及短期结果如何转化为长期后果。

## 2. 核心循环一句话定义

玩家在每个 20 分钟回合中，面对由角色和事件施加的职场压力，通过聊天对话中的有限应对选项或自由输入处理冲突，并承担由资源消耗、隐患累积、风险上升和污染变化带来的长期后果。

## 3. 基础循环结构

```text
时间推进
-> 既有状态与隐患结算
-> 渲染状态栏
-> 抽取角色与事件
-> 输出事件摘要与可选行动
-> 用户回复编号 / 关键词 / 自然语言
-> 系统归类并结算资源与后果
-> 生成后续链条
-> 进入下一回合
```

## 4. 玩家每回合面对的核心问题

玩家每回合都要在以下目标之间进行取舍：

- 保住精力与体力
- 保住绩效
- 降低风险
- 避免生成隐患
- 减少被组织盯上的概率
- 尽量不让污染值快速上升

这意味着《暗黑职场》的核心不是“赢一场战斗”，而是“在没有完美答案的情况下活下来”。

## 5. 循环中的关键输入

系统在每回合施加压力时，主要受以下输入影响：

- 当前时间段
- 当前状态卡
- 当前隐患卡
- 当前项目区压力
- 当前全局机制
- 角色权重
- 最近发生的历史事件

## 6. 循环中的关键输出

一次应对结算后，系统可能输出：

- 核心数值变化
- 骰子判定结果档位
- 新状态卡
- 新隐患卡
- 新项目压力
- 新后续事件链
- 角色态度变化
- 失败或阶段结束判定

## 7. 短期循环与长期循环

### 7.1 短期循环

短期循环聚焦单回合冲突：

- 当前发生了什么
- 用户在聊天中选择了什么应对
- 当回合付出了什么代价

### 7.2 长期循环

长期循环聚焦累积后果：

- 低精力是否侵蚀生命上限
- 隐患是否即将引爆
- 风险是否逼近暴雷阈值
- 污染是否改变玩家路线与结局倾向
- KPI 是否逼近开除线

短期最优选择未必是长期最优选择，这是本项目的核心张力来源。

## 8. 典型循环样例

### 8.1 推活链条

```text
同事施压
-> 玩家口头答应
-> 当回合压力下降
-> 生成“责任未明确”隐患
-> 数回合后任务出错
-> 玩家承担背锅后果
```

### 8.2 硬扛链条

```text
甲方加急
-> 玩家熬夜硬扛
-> 当回合保住 KPI
-> 获得“透支”状态
-> 次日高压事件额外掉 EN/ST
-> 更易触发翻车事件
```

### 8.3 灰色合作链条

```text
财务或流程事件施压
-> 玩家接受灰色处理
-> 当回合短期收益
-> RISK 和 COR 上升
-> 后续审计或问责事件权重上升
-> 暴雷时承受更大损失
```

## 9. 失败压力与生存目标

玩家的长期目标不是最大化单一资源，而是在多条失败线之间求平衡：

- `HP <= 0`：崩溃或出局
- `KPI <= 0`：被开除
- 高风险长期累积：触发暴雷类灾难
- 高污染持续累积：推进黑化路线并改变结局倾向

## 10. 设计原则

核心循环应持续满足以下原则：

- 每回合都要有真实压力
- 玩家始终面对代价明确的选择
- 资源变化要能映射到现实职场感受
- 后果要能延迟爆发，而不是只在当回合结算完毕
- 不追求完美解，追求生存感与讽刺感

## 11. 随机性原则

《暗黑职场》采用“默认确定性结算，关键节点触发骰子判定”的混合模式。

设计原则：

- 普通回合以确定性规则为主，保证节奏稳定
- 只有本来就具有高度不确定性的场景才触发骰子
- 用户的准备、状态和选项决定底盘，骰子决定结果质量
- 骰子优先影响“成功质量”和“附加后果”，而不是覆盖策略价值
- 留痕、需求确认、边界管理等准备动作必须能改善判定

## 11. 聊天式交互约束

在 Agent 或飞书等聊天载体中，核心循环应遵循以下硬规则：

- 每轮对话最先显示用户状态栏
- 默认提供 3 到 5 个明确可执行选项
- 用户既可以回复编号，也可以自由输入自然语言
- 系统优先匹配明确选项，匹配不到时再进行意图归类
- 不要求用户理解内部卡牌结构，也能顺畅推进

推荐的对话顺序：

```text
状态栏
-> 事件摘要
-> 风险提示
-> 可选行动
-> 用户回复
-> 结算反馈
-> 下一轮状态栏
```
