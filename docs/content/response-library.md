# 应对库规划

## 1. 文档目的

本文档定义《暗黑职场》第一版应对库的结构规划、掷骰触发条件与优先补卡方向。

## 2. 应对库目标规模

第一版建议：

- 完整目标：18 到 24 张应对卡
- 原型可玩下限：10 到 12 张应对卡

## 3. 应对库组织方式

建议按两层组织：

1. 按应对类型组织
2. 按是否触发掷骰组织

## 4. 应对类型清单

- 执行类
- 留痕类
- 规避类
- 谈判类
- 转移类
- 恢复类
- 政治类

## 5. 掷骰触发规则

### 5.1 默认不掷骰的应对

以下类型默认按确定性规则结算：

- 普通执行
- 常规留痕
- 常规恢复
- 低风险边界澄清

### 5.2 触发掷骰的应对

以下类型通常属于 `high_risk_action`，需要触发 `2d6 + 修正值`：

- 温和拒绝
- 明确拒绝
- 向上汇报
- 转移责任
- 灰色合作
- 公开站队
- 硬扛加急

### 5.3 条件触发掷骰的应对

以下应对通常使用 `conditional`：

- 缩小范围
  说明：在普通任务中可确定性结算；在高压上司、甲方加急、关键节点时触发掷骰
- 拖延规避
  说明：只有在存在截止压力或关键追责时触发掷骰
- 需求确认
  说明：通常不掷骰，但若对抗强压角色或已处于隐患翻面阶段，可触发掷骰

## 6. 推荐首批应对卡清单

### 6.1 确定性优先卡

- 立即执行
- 邮件留痕
- 需求确认
- 午休
- 睡觉
- 健身
- 边界重申

### 6.2 高风险掷骰卡

- 温和拒绝
- 熬夜硬扛
- 缩小范围
- 向上汇报
- 转移责任
- 公开站队

## 7. 应对卡建设要求

- 每张应对卡必须明确主要收益
- 每张应对卡必须明确主要代价
- 会触发掷骰的应对卡必须明确触发类型与四档结果
- 同类应对卡之间要有真实策略差异，而不是只换文案

## 8. 推荐判定映射

| 应对卡 | 默认触发类型 | 说明 |
| --- | --- | --- |
| 立即执行 | `none` | 基础执行动作 |
| 邮件留痕 | `none` | 主要是风险管理 |
| 需求确认 | `conditional` | 对抗强压时可能进入判定 |
| 缩小范围 | `conditional` | 关键交付场景下需要判定 |
| 温和拒绝 | `required` | 典型高风险谈判行为 |
| 向上汇报 | `required` | 容易引发角色态度波动 |
| 转移责任 | `required` | 高污染高风险动作 |
| 熬夜硬扛 | `required` | 通常能成，但结果质量与代价应带波动 |
