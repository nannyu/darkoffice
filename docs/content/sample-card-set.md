# 首批样测牌组建议

## 1. 文档目的

本文档给出首批可用于原型验证的样测牌组规模建议，用于指导后续内容填充顺序。

## 2. P0 样测牌组

- 角色：6 个
- 事件：20 到 30 张
- 应对：10 到 12 张
- 状态：8 张左右
- 隐患：6 张左右
- 机制：4 张左右

目标：先跑通一个最小可验证循环。

## 3. 推荐优先角色

- PUA 上司
- 推活同事
- 甲方金主
- HR 笑面虎
- 财务 / 采购关键人
- 派系总监

## 4. 推荐优先事件方向

- 加急任务
- 模糊责任
- 留痕对抗
- 绩效问责
- 灰色流程
- 站队表态

其中建议优先纳入掷骰验证的事件：

- 加急任务
- 绩效问责
- 灰色流程
- 站队表态

## 5. 推荐优先应对方向

- 立即执行
- 邮件留痕
- 缩小范围
- 温和拒绝
- 熬夜硬扛
- 需求确认
- 拖延规避
- 摸鱼恢复
- 转移责任
- 边界重申

其中建议优先纳入掷骰验证的应对：

- 温和拒绝
- 缩小范围
- 熬夜硬扛
- 转移责任
- 向上汇报

## 6. 样测牌组使用原则

- 优先验证规则差异，不优先追求数量
- 优先覆盖核心体验，不优先追求剧情多样
- 每张卡都应承担测试价值

## 7. P0 掷骰覆盖要求

为验证骰子系统是否必要且稳定，P0 样测牌组建议至少覆盖：

- `required` 事件 4 张以上
- `conditional` 事件 4 张以上
- `required` 应对 4 张以上
- `conditional` 应对 3 张以上
