# 事件生成规则

## 1. 文档目的

本文档定义《暗黑职场》中角色与事件的抽取逻辑、权重因子与链式生成原则。

## 2. 生成目标

事件生成系统需要同时满足：

- 保持每回合都有真实压力
- 让压力来源与当前状态相关，而不是完全随机
- 支持短期波动与长期连锁
- 避免机械重复

## 3. 生成流程

正式生成顺序如下：

1. 读取当前时间段
2. 读取状态区、隐患区、项目区与机制区
3. 计算来访角色权重
4. 抽取本回合角色
5. 根据角色事件池与全局修正抽取主事件
6. 视情况追加副事件
7. 将结果送入玩家应对阶段

## 4. 角色抽取权重因子

| 因子 | 影响说明 |
| --- | --- |
| 当前时间段 | 晚间更容易触发上司或甲方 |
| KPI 水平 | 低 KPI 更容易触发 HR |
| RISK 水平 | 高 RISK 更容易触发财务、审计或总监相关压力 |
| COR 水平 | 高 COR 更容易触发政治型事件 |
| 当前状态 | 被盯上等状态会提高敌对事件概率 |
| 项目区内容 | 有客户项目时甲方更容易出现 |
| 隐患区内容 | 报销、流程类隐患会提高财务或采购相关角色概率 |
| 历史关系 | 得罪过某角色时，其再出现概率上升 |

示意公式：

`最终角色权重 = 基础权重 × 时间修正 × 状态修正 × 项目修正 × 隐患修正 × 机制修正`

## 5. 事件抽取权重因子

| 因子 | 影响说明 |
| --- | --- |
| 来源角色 | 决定事件池范围 |
| 当前压力等级 | 高压阶段提高高强度事件概率 |
| 当前机制 | 如绩效季提高问责类事件 |
| 当前状态 | 低 EN 更易触发崩溃型事件 |
| 最近历史 | 用于防止完全重复，也可制造连续追击 |

## 6. 角色与事件池映射原则

不同角色应具备明确的事件偏好：

- 上司：任务类、压榨类、问责类
- 同事：推活类、甩锅类、边界模糊类
- HR：绩效类、制度类、态度类
- 甲方：需求变更类、临时加急类
- 财务或采购：流程类、合规类、倒签类
- 总监：站队类、汇报类、政治类

## 7. 副事件生成原则

副事件不是常规配置，只在以下场景触发：

- 高压时间段叠加
- 某些机制卡生效
- 玩家当前状态极差
- 某角色被动明确要求追加压力

副事件的目标是形成“被连续围堵”的压迫感，而不是无差别增加复杂度。

## 8. 链式生成原则

事件处理结果可能生成：

- 新状态
- 新隐患
- 新项目压力
- 新后续事件

链式生成的核心原则：

- 短期缓解应允许长期反噬
- 不留痕、口头承诺、灰色合作应优先进入隐患链
- 组织政治类选择应优先进入关系与态度链
- 高风险事件应优先进入暴雷链

## 9. 反重复原则

为了避免体验单调，事件生成应遵循：

- 不连续过多重复同一张事件卡
- 相同角色连续出现时，应优先切换事件类型
- 允许重复主题，但应改变上下文或后果
- 在高压阶段，重复是设计工具，但不能变成机械刷脸

## 10. 当前约束

- 当前只定义规则，不定义完整事件库
- 完整角色池与事件池应在内容文档中维护
- 新增角色或事件前，必须先说明其进入哪个事件池、影响哪类权重
