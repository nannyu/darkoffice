# 卡牌系统

## 1. 文档目的

本文档定义《暗黑职场》的卡牌系统边界、卡牌分类、基础牌区、通用字段与类型级规则。

## 2. 系统目标

卡牌系统负责将“恶性职场”的压力与应对方式转化为可操作、可结算、可扩展的规则对象。

它需要同时支持：

- 事件施压
- 玩家应对
- 状态持续生效
- 隐患延迟爆炸
- 项目长期施压
- 机制卡提供全局修正

## 3. 卡牌系统在整体中的位置

卡牌系统不是独立小游戏，而是系统层的核心表达方式。它连接了：

- 角色系统
- 事件系统
- 数值系统
- 回合流程
- 内容生产模板

## 4. 卡牌分类

当前正式卡牌类型如下：

| 枚举值 | 中文 | 作用 |
| --- | --- | --- |
| `CHARACTER` | 人物卡 | 定义本回合施压来源 |
| `EVENT` | 事件卡 | 定义本回合主冲突 |
| `RESPONSE` | 应对卡 | 定义玩家处理方式 |
| `STATUS` | 状态卡 | 定义持续效果 |
| `HAZARD` | 隐患卡 | 定义延迟爆炸问题 |
| `MECHANIC` | 机制卡 | 定义全局环境修正 |
| `PROJECT` | 项目卡 | 定义持续任务压力 |

## 5. 基础牌区

| 牌区名称 | 功能 | 是否常驻 | 说明 |
| --- | --- | --- | --- |
| 来访角色区 | 显示当前施压角色 | 是 | 本回合压力来源 |
| 当前事件区 | 显示主事件卡 | 是 | 本回合核心冲突 |
| 手牌区 | 显示当前可用应对卡 | 是 | 玩家操作核心区域 |
| 状态区 | 显示持续状态 | 是 | 持续增减益结算区 |
| 隐患区 | 显示未爆炸问题 | 是 | 倒计时与翻面核心区域 |
| 项目区 | 显示持续工作任务 | 是 | 长线压力来源 |
| 历史记录区 | 记录已发生卡牌 | 否 | 用于回顾与追溯 |

结算优先级建议固定为：

`状态区 -> 隐患区 -> 项目区 -> 角色区 -> 事件区 -> 手牌区 -> 历史区`

## 7. 类型级定义

### 7.1 人物卡

人物卡定义施压来源与角色被动，不直接等同于事件本身。

人物卡至少应包含：

- 基础身份信息
- 角色标签
- 事件池关联
- 被动修正
- 与玩家关系或态度修正

### 7.2 事件卡

事件卡定义本回合主冲突，是系统施压的核心载体。

事件卡应覆盖：

- 来源角色
- 压力类型
- 基础后果
- 可选应对限制
- 是否生成后续链条

### 7.3 应对卡

应对卡定义玩家处理事件的方式。它不是“攻击技能”，而是职场行为策略。

应对卡应覆盖：

- 资源消耗
- 风险与污染变化
- 对事件结果的减免、替换或转移
- 是否可能生成新的状态、隐患或角色态度变化

### 7.4 状态卡

状态卡定义持续效果，例如疲惫、透支、被盯上、领导满意等。

状态卡要明确：

- 生效时机
- 持续回合
- 是否叠加
- 消除条件

### 7.5 隐患卡

隐患卡定义延迟爆炸的问题，是系统长期压力的重要组成部分。

隐患卡要明确：

- 倒计时
- 翻面条件
- 爆炸效果
- 清理方式

### 7.6 机制卡

机制卡定义全局环境修正，例如绩效季、审计抽查、组织调整等。

机制卡通常会：

- 修改事件概率
- 修改结算修正
- 为一段时期增加共同压力

### 7.7 项目卡

项目卡定义持续任务压力，用来承载中长期工作负担。

项目卡要明确：

- 槽位占用
- 剩余进度或剩余步骤
- 每回合压力
- 截止时点
- 成败后果

## 8. 稀有度与标签

当前可用稀有度建议：

| 值 | 含义 |
| --- | --- |
| `COMMON` | 常见 |
| `UNCOMMON` | 不常见 |
| `RARE` | 稀有 |
| `EPIC` | 高稀有 |
| `BOSS` | 关键牌 |

标签用于支持筛选、抽取与联动，不承担规则定义主责。正式规则应写在字段或结算结构中，而不是只藏在标签里。

## 9. 设计约束

- 卡牌必须服务于核心循环，而不是只追求数量堆叠
- 同一张卡只承载一个清晰的系统职责
- 卡牌描述文本与结构化字段不得互相矛盾
- 能用结构化字段表达的规则，不只写自然语言
- 事件与应对的收益必须伴随代价，避免无成本最优解
