# 状态、隐患与项目系统

## 1. 文档目的

本文档定义《暗黑职场》中状态卡、隐患卡与项目卡的系统职责与交互关系。

## 2. 三者区别

| 系统 | 核心作用 | 时间特征 | 典型表现 |
| --- | --- | --- | --- |
| 状态 | 持续效果 | 短中期持续 | 疲惫、透支、被盯上 |
| 隐患 | 延迟爆炸问题 | 倒计时后翻面 | 责任未明确、倒签文件 |
| 项目 | 持续任务压力 | 中长期推进 | 季度汇报、客户对接 |

## 3. 状态系统

### 3.1 职责

状态系统用于表示玩家当下处于何种持续影响之中，并在每回合结算中提供修正。

### 3.2 示例状态

- 疲惫
- 透支
- 被盯上
- 领导满意
- 领导不满
- 站队嫌疑
- 合规风险
- 心灰意冷

### 3.3 设计要求

- 必须明确持续回合
- 必须明确结算时机
- 必须明确移除条件
- 需要说明是否可叠加

## 4. 隐患系统

### 4.1 职责

隐患系统用于表示“暂时没爆炸，但迟早要算”的问题。

### 4.2 示例隐患

- 口头承诺
- 责任未明确
- 需求未确认
- 报销缺材料
- 倒签文件
- 未回消息
- 周报未交

### 4.3 设计要求

- 必须有倒计时
- 必须定义归零触发结果
- 必须定义清除方式
- 可定义被哪些角色或机制放大

## 5. 项目系统

### 5.1 职责

项目系统用于承载持续任务压力，是长期工作负担的主要对象。

### 5.2 典型表现

- 占用槽位
- 每回合施加压力
- 临近截止时压力加重
- 完成后提供奖励，失败后追加惩罚

## 6. 三者之间的联动

- 状态会放大事件与数值损失
- 隐患会在未来翻面为新事件或强制惩罚
- 项目会持续制造每回合压力，并提高某些角色出现概率
- 项目执行不当可能生成隐患
- 隐患爆炸可能附加状态

## 7. 设计原则

- 状态负责“现在正在难受什么”
- 隐患负责“以后会出什么事”
- 项目负责“长期压着你的是什么”
- 三者必须相互联动，避免各自独立成为孤立子系统
