# 数值与资源系统

## 1. 文档目的

本文档定义《暗黑职场》的核心数值、资源阈值、上限规则与基础变化规律。

## 2. 核心数值总表

| 缩写 | 名称 | 初始值 | 最低值 | 默认上限 | 失败条件 | 系统定位 |
| --- | --- | ---: | ---: | ---: | --- | --- |
| `HP` | 生命值 | 100 | 0 | 100 | `HP <= 0` 崩溃或出局 | 总体生存线 |
| `EN` | 精力值 | 100 | 0 | 100 | 间接侵蚀 HP | 主消耗资源 |
| `ST` | 体力值 | 100 | 0 | 100 | 间接影响恢复 | 生理资源 |
| `KPI` | 绩效值 | 100 | 0 | 100 | `KPI <= 0` 被开除 | 组织容忍度 |
| `RISK` | 风险值 | 0 | 0 | 无硬上限 | 高风险触发暴雷 | 延迟惩罚累积 |
| `COR` | 污染值 | 0 | 0 | 无硬上限 | 决定黑化倾向 | 道德异化程度 |

## 3. 数值职责

| 数值 | 主要来源 | 主要变化来源 | 对后续系统影响 |
| --- | --- | --- | --- |
| `HP` | 少量恢复事件 | 长期低 EN、重大打击、暴雷 | 决定 EN 上限 |
| `EN` | 休息、恢复卡 | 任务、沟通、压榨、焦虑 | 最核心即时资源 |
| `ST` | 睡眠、规律恢复 | 熬夜、奔波、健身、硬扛 | 决定恢复质量 |
| `KPI` | 完成任务、领导满意 | 失败、背锅、拒绝、站错队 | 决定是否被开除 |
| `RISK` | 违规、无留痕、隐患 | 清理隐患、留痕、规范处理 | 决定高危事件概率 |
| `COR` | 甩锅、权术、灰色合作 | 少量净化事件 | 决定路线与结局倾向 |

## 4. 上限规则

| 上限项 | 初始值 | 规则 |
| --- | ---: | --- |
| `HP_MAX` | 100 | 长期低 EN 可下降 |
| `EN_MAX` | 100 | 不得超过 `HP_MAX` |
| `ST_MAX` | 100 | 长期透支可下降 |

### 4.1 生命上限下降规则

- 若 `EN < 30` 连续 3 回合，则 `HP_MAX -2`
- 若处于“透支”状态且再次使用高消耗动作，则 `HP_MAX -1`
- 若发生重大暴雷事件，则 `HP_MAX -3` 到 `-10`

### 4.2 精力上限规则

`EN_MAX = min(100, HP_MAX)`

### 4.3 体力上限规则

- 若连续两天深夜加班且睡眠不足，则 `ST_MAX -2`
- 若连续执行“健身 + 正常睡眠”等恢复行为，可恢复 `ST_MAX`

## 5. 阈值状态

| 数值区间 | 状态含义 | 系统影响 |
| --- | --- | --- |
| `EN >= 70` | 精神稳定 | 正常发挥 |
| `30 <= EN < 70` | 正常疲劳 | 无额外惩罚 |
| `10 <= EN < 30` | 低精力 | 执行质量下降，生命上限受威胁 |
| `EN < 10` | 濒临崩溃 | 高危事件额外伤害 |
| `ST < 30` | 低体力 | 恢复效率下降 |
| `KPI < 40` | 危险绩效 | HR 与上司事件权重上升 |
| `RISK >= 50` | 高风险 | 暴雷类事件权重上升 |
| `COR >= 50` | 高污染 | 解锁黑化路线卡，部分结局锁死 |

## 9. 设计约束

- `EN` 应作为最常用即时资源
- `HP` 不应频繁直接掉落，而应体现长期侵蚀
- `KPI` 不等于能力，而是组织视角下的生存资格
- `RISK` 负责延迟爆炸
- `COR` 负责价值异化与路线分化
