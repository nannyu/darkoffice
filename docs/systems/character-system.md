# 角色系统

## 1. 文档目的

本文档定义《暗黑职场》中的角色系统边界、角色类型、角色被动与态度变化规则。

## 2. 角色系统职责

角色系统负责提供：

- 压力来源的人格化载体
- 不同事件池的入口
- 特定被动修正
- 玩家关系与态度变化
- 职场生态中的派系与利益结构

## 3. 设计原则

- 角色坏，但要坏得合理
- 每个角色都应有利益动机和惯用手法
- 角色不是脸谱化吼叫工具，而是系统压力的不同表现形式
- 制度比个人更坏，角色应体现制度如何落到个体身上

## 4. 当前推荐角色池

第一版推荐至少包含以下角色：

1. 黑心老板
2. PUA 直属上司
3. 笑里藏刀老油条
4. 推活同事
5. 派系总监
6. 关系户
7. 财务 / 采购关键人
8. 乙方供应商
9. 甲方金主
10. HR 笑面虎

## 5. 角色字段要点

角色卡正式字段模板见 [card-templates.md](/Users/niunan/project/darkoffice/docs/content/card-templates.md)，当前系统层重点关注：

- 角色 ID
- 职位或角色类型
- 派系
- 性格标签
- 被动效果
- 偏好事件标签
- 初始敌意与好感
- 说话风格
- 特殊联动规则

## 6. 角色被动框架

当前规则层推荐的角色被动如下：

| 角色 | 被动效果 |
| --- | --- |
| 黑心老板 | “公司大局”“成长机会”类事件额外压力 +1 |
| PUA 上司 | 玩家使用“温和拒绝”时，额外 `KPI -3` |
| 老油条 | 若玩家当前没有留痕保护，本回合生成隐患概率提升 |
| 推活同事 | 玩家若选择“立即执行”，更容易追加次级任务 |
| 派系总监 | 若玩家有“站队嫌疑”，其事件强度提升 |
| 关系户 | 其造成的项目失误更难通过正常沟通解决 |
| 财务 / 采购关键人 | 灰色操作收益更大，但审计反噬更重 |
| 乙方供应商 | 口头承诺类隐患更容易生成 |
| 甲方金主 | 加急类任务自带倒计时压力 |
| HR 笑面虎 | 玩家 KPI 越低，其事件权重越高 |

## 7. 角色态度规则

角色态度用于记录角色与玩家关系的动态变化，建议至少包含：

- 敌意
- 满意度
- 好感
- 站队倾向

角色态度可由以下行为触发变化：

- 留痕或拒绝使部分角色敌意上升
- 硬扛任务使上司或甲方满意度上升
- 公开站队提升某派系好感，同时增加其他派系敌意
- 灰色合作可能提高短期合作评价，但增加长期风险

## 8. 角色与派系

派系不是所有角色的强制字段，但在组织政治相关角色中必须明确。

派系的主要作用：

- 提供站队事件基础
- 影响角色权重修正
- 决定部分后续事件链方向

## 9. 角色写作要求

- 角色语言要体现权力位置与说话风格
- 同一角色在不同场景下可以变体，但不能失去核心手法
- 角色台词应压迫、虚伪、暧昧或冷漠，但不应只靠粗暴辱骂完成表达
