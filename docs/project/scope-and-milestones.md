# 范围与里程碑

## 1. 文档目的

本文档定义《暗黑职场》项目当前阶段的范围边界、推进顺序与阶段性里程碑。由于技术栈尚未定案，本文档采用阶段里程碑，而非日历排期。

## 2. 当前阶段结论

当前项目处于文档建设阶段，核心目标不是启动开发，而是建立稳定、可维护、适合 AI/Agent 与人协作的事实源文档体系。

## 3. 当前范围

### 3.1 当前必须完成的事项

- 建立文档导航与事实源优先级
- 将历史总案拆分为结构化子文档
- 统一术语、规则边界与文档命名
- 固化核心循环、卡牌系统、回合流程与模板规则
- 明确开发规范与 AI/Agent 协作方式

### 3.2 当前明确不做的事项

- 技术栈定案
- 正式工程初始化
- UI 设计定稿
- 可运行原型开发
- 大规模内容生产

## 4. 里程碑定义

### M0：文档骨架建立

目标：

- 建立 `README`、`docs/INDEX` 与协作规范
- 明确各文档域职责
- 保留历史文档，但完成降级与映射

完成标准：

- 文档目录结构稳定
- 团队能明确知道“哪个问题看哪份文档”

### M1：核心事实源拆分

目标：

- 从历史总案中拆出项目定位、核心循环、卡牌系统、回合流程、卡牌模板等核心文档

完成标准：

- 至少形成一批可独立引用的主题文档
- 新增共识不再回写到历史总案

### M2：规则消歧与结构化

目标：

- 清理重复定义
- 对关键术语、字段、规则和流程进行结构化表达

完成标准：

- 核心文档之间无明显冲突
- 关键系统可以用明确字段和流程表达

### M3：内容生产约束成型

目标：

- 建立角色、事件、状态、隐患、项目等内容模板
- 建立文本调性与写作约束

完成标准：

- 内容创作不再依赖自由发挥
- 后续扩写可以在统一模板下推进

### M4：技术预研前评审

目标：

- 评估文档是否足以支撑首个原型定义
- 决定是否进入技术预研

完成标准：

- 能明确回答原型形态、核心闭环、最小实现范围和验证方法

## 5. 当前推荐推进顺序

1. `project/positioning.md`
2. `design/core-loop.md`
3. `systems/card-system.md`
4. `systems/turn-flow.md`
5. `systems/stats-and-resources.md`
6. `content/card-templates.md`
7. `content/tone-and-writing.md`
8. `project/glossary.md`

## 6. 当前阶段验收标准

当前阶段结束前，至少应满足：

- 项目定位、范围和核心循环有正式事实源
- 卡牌系统、回合流程和数值规则有正式事实源
- 内容模板与协作规范可供人和 AI/Agent 直接引用
- 历史文档不再承担“唯一权威文档”职责

## 7. 范围管理规则

- 新增工作前，先判断是否属于当前里程碑
- 若不属于当前里程碑，应记录为后续议题，不强行并入
- 当某项议题会引入技术决策时，默认延期到技术预研阶段讨论
