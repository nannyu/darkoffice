## 4. 里程碑定义

### M0：文档骨架建立 ✅

目标：

- 建立 `README`、`docs/INDEX` 与协作规范
- 明确各文档域职责
- 保留历史文档，但完成降级与映射

完成标准：

- 文档目录结构稳定
- 团队能明确知道“哪个问题看哪份文档”

### M1：核心事实源拆分 ✅

目标：

- 从历史总案中拆出项目定位、核心循环、卡牌系统、回合流程、卡牌模板等核心文档

完成标准：

- 至少形成一批可独立引用的主题文档
- 新增共识不再回写到历史总案

### M2：规则消歧与结构化 ✅

目标：

- 清理重复定义
- 对关键术语、字段、规则和流程进行结构化表达

完成标准：

- 核心文档之间无明显冲突
- 关键系统可以用明确字段和流程表达

### M3：内容生产约束成型 ✅

目标：

- 建立角色、事件、状态、隐患、项目等内容模板
- 建立文本调性与写作约束

完成标准：

- 内容创作不再依赖自由发挥
- 后续扩写可以在统一模板下推进

### M4：技术预研前评审 ✅

目标：

- 评估文档是否足以支撑首个原型定义
- 决定是否进入技术预研

完成标准：

- 能明确回答原型形态、核心闭环、最小实现范围和验证方法

---

### M5：最小运行闭环 ✅ **已完成**

目标：

- 建立可持久化的最小运行闭环（创建 → 回合 → 结算 → 落盘 → 查看）
- 技术栈定案并落地

完成标准：

- `runtime/db.py`：SQLite schema + 初始化 + 迁移
- `runtime/engine.py`：回合结算引擎（含状态机、判定、隐患、项目）
- `scripts/game_state_cli.py`：CLI 入口（init/create/turn/show）
- `skill/darkoffice-persistent-skill.md`：可交付 Agent Skill
- 验证命令：`python scripts/game_state_cli.py init && create demo && turn demo --action EMAIL_TRACE --mod 3 && show demo`

### M6：素材库与内容扩展 ✅ **已完成**

目标：

- 建立素材库系统，支持外部案例导入
- 建立卡牌蒸馏器，AI 从素材生成角色卡/事件卡/隐患卡
- 建立剧情线系统，支持多幕式剧情编排

完成标准：

- `runtime/materials.py`：素材 CRUD + 文件导入(md/txt/pdf) + 自定义卡牌管理
- `runtime/storylines.py`：剧情线 CRUD + 激活/推进/完成
- `scripts/distill_template.py`：蒸馏提示词模板 + JSON Schema 校验
- 素材库已导入 19 条中央纪委"警钟"栏目反腐案例
- 剧情线引擎支持剧情模式（优先抽取剧情线指定角色/事件）

### M7：内容规模化生产 **当前阶段**

目标：

- 从素材库蒸馏生成更多角色卡、事件卡、隐患卡
- 编排至少 1 条完整剧情线
- 平衡性调优与数值验证

完成标准：

- 自定义卡牌数量 ≥ 50 张（角色+事件+隐患）
- 至少 1 条剧情线可完整运行（≥3 幕）
- `scripts/simulate_balance.py` 可输出生存率统计
- 代码审计报告中的问题全部修复

## 7. 范围管理规则

- 新增工作前，先判断是否属于当前里程碑
- 若不属于当前里程碑，应记录为后续议题，不强行并入
- 当某项议题会引入技术决策时，默认延期到技术预研阶段讨论
