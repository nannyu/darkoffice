# 项目定位

## 1. 文档目的

本文档定义《暗黑职场》项目的产品定位、核心体验、目标载体与当前阶段共识，是项目层事实源之一。

## 2. 项目定义

《暗黑职场》是一个以恶性职场为主题的卡牌驱动叙事生存项目。

玩家扮演刚进入社会的新人，在一家公司中面对压榨、甩锅、留痕、站队、绩效规训与灰色合规，通过卡牌系统进行应对，在生存、自保、妥协与黑化之间做选择。

## 3. 产品定位

### 3.1 类型定位

- 叙事驱动
- 卡牌驱动
- 回合制
- 生存管理
- 黑色职场讽刺

### 3.2 核心表达

本项目不是传统战斗游戏，也不是纯剧情选项游戏。核心冲突来自职场压力的系统化实体化：

- 任务压力
- 责任甩锅
- 情绪压榨
- 组织政治
- 合规风险
- 资源透支

### 3.3 目标载体

当前文档阶段认可的目标载体包括：

- Agent 对话原型
- 飞书聊天对话原型
- 文字游戏原型
- 多 Agent 协作原型

当前优先交互形态为“聊天对话推进”，而不是传统桌面卡牌界面。

是否进一步扩展为 Web、桌面应用或其他图形化形态，暂未定案。

## 4. 玩家角色与基础处境

玩家的起始身份是新人或普通员工，具备以下初始处境：

- 没背景
- 没派系
- 没资源
- 没话语权
- 仍保留一定理想主义

这意味着玩家前期天然弱势，必须依赖策略、留痕、取舍与风险管理求生。

## 5. 核心体验目标

项目应稳定传达以下体验：

1. 被持续压迫的生存感
2. 资源持续衰减的焦虑感
3. 以策略而非蛮力求生的博弈感
4. 职场潜规则被实体化后的识别感
5. 存活与被同化之间的价值冲突

## 6. 核心设计目标

第一版系统必须满足以下目标：

- 通过卡牌而非纯弹窗推动事件
- 形成“角色施压 -> 玩家应对 -> 状态累积 -> 连锁后果”的稳定循环
- 让资源、隐患、风险、污染和绩效共同构成长期压力
- 让系统规则清晰，支持后续扩展角色、事件与内容资产

## 7. 当前版本边界

### 7.1 当前版本覆盖范围

当前文档共识覆盖：

- 单人玩法
- 回合制流程
- 核心数值系统
- 卡牌分类与字段结构
- 基础牌区
- 回合状态机
- 基础结算顺序

### 7.2 当前版本明确不覆盖

当前阶段不纳入正式范围：

- 复杂 AI 对抗演出
- 复杂构筑成长树
- 全剧情分支树
- 美术终稿与 UI 终稿
- 多周目 meta 养成
- 联机与多人模式

## 8. 成功标准

当项目进入首个可运行原型阶段时，应至少满足以下条件：

- 玩家能在连续数个回合中感受到明确的压力循环
- 关键资源之间存在真实取舍，而非单一最优解
- 事件处理能产生后续状态、隐患或风险后果
- 文本体验能够传达职场讽刺，而不是抽象数值变化
