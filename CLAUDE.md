# CLAUDE.md

## 项目目标

`darkoffice` 当前目标是把《暗黑职场》从文档资产推进到“可运行、可复盘、可持久化”的 Agent Skill 形态。

## 约束优先级

1. 用户体验优先于技术偏好
2. 单一事实源优先于聊天结论
3. 先最小闭环，后扩展功能
4. 先验证，后宣称完成

## 仓库结构约定

- `docs/`：正式文档事实源
- `skill/`：可交付的 Agent Skill 文件
- `runtime/`：数值规则执行与数据存取逻辑
- `scripts/`：初始化、调试、运维脚本
- 根目录仅保留入口文件，不堆临时草稿

## 开发规则

- 新增能力时，先确认属于哪个文档域
- 任何规则变更，先更新 `docs/` 对应事实源，再改实现
- 不直接改写历史归档文档作为“当前规则”
- 不引入密钥或凭据到代码与文档

## 持久化与数值维护规则

- 默认使用 SQLite 文件作为本地持久化
- 任意回合结算必须写入 `turn_logs`
- 任意关键状态变更必须更新 `game_sessions`
- 所有数值变更使用结构化 `delta` 记录

## 验证底线

- 修改后至少跑一次可重复命令验证（init/create/apply/show）
- 若验证失败，不允许通过注释错误绕过
- 对外声明“完成”必须包含可复现命令
